 Linux Documentation Project: Authors - Contribute / Help : Unmaintained 

We must admit that a lot of the documents in the LDP collection are outdated. Yet they are not deleted, partly out of historical interest, partly because there are still people referencing these documents and actively using their content. Still, a document that is not updated for over a year should be read with caution, as we consider it outdated. Such documents are eligible for a change of ownership.

If you wish to become the "owner" of one of our documents, please follow these steps:

  1. Contact the original author. If not listed, consult the document. Make sure the author no longer wishes to maintain the document in question. Note that the email address shown may no longer be valid. In that case, please try a search for the author. If they cannot be found after a "good-faith" effort, let us know (feedback@en.tldp.org).

  2. Determine if a more up-to-date copy of the document exists: there might be a newer HOWTO covering the old document, or you might find a new version outside of the LDP. If the latter is the case, secure a copy for yourself to work on. Should you find that there is a newer HOWTO in our collection, but the old HOWTO does not point in a sufficiently clear way to the new document, please let us know.

  3. Inform the LDP which document you would like to own, so that we can track who is working on what and prevent duplication of effort. We also suggest that you join the LDP general discussion list (discuss@en.tldp.org).

  4. Submit the document to the LDP with any intended modifications. Make sure to continue to reference the original author somewhere within the document (Credits, Revision History, etc.). Once the document is re-submitted, we will remove the entry from the list below.
