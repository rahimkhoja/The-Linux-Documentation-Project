Configuración especial en /etc/inittab

El archivo /etc/inittab tiene algunas características particulares que permiten a init reaccionar ante circunstancias especiales. Estas características son definidas por palabras claves en el tercer campo. Algunos ejemplos:

powerwait

Le indica a init que apague al sistema ante una falla en el suministro eléctrico. Esta característica asume el uso de una UPS, y de un software que examina la UPS e informa a init cuando exista un corte de energía.

ctrlaltdel

Le encomienda a init que reinicie el sistema cuando el usuario presione ctrl-alt-del en el teclado de la consola. Note que el administrador del sistema puede configurar la reacción de ctrl-alt-del para que realice alguna acción distinta, como por ejemplo, ignorar el evento, si el sistema se encuentra en una ubicación pública (o para iniciar nethack).

sysinit

El comando a ser ejecutado cuando el sistema es iniciado. Este comando usualmente limpia el directorio /tmp, por ejemplo.

La lista anterior no está completa. Lea la página de manual de inittab para conocer todas las posibilidades, y detalles de las palabras claves anteriores.
