为了加速文件处理的速度和保持 LinuxFocus 文章的品质及完整性， 我们列出了以下几点守则，希望想要投稿的作者们能够参考并尽量使用这些统一的格式。





写作守则

  • 我们建议您用文字编辑器来编辑您的文章，虽然您也可以用一些『所见即所得』 WYSIWYG 的 HTML 编辑器来写作，但这些编辑器也常在档案内加人一些 "紊乱" 的 HTML 语法，如一大堆的 &nbsp; 和 <BR> 等等。
  • 不管您决定用什麽编辑您的文章您都应该使用我们的样本(template.meta.html)。这个档案只是一个简单的 HTML 档案但包函了一些结构信息。您只要把文章放在 ArticleBody 这一个关键词的後面就可以了。
  • 如果文章的主题需要，我们建议作者们在文章中加入一些插图或图片， 作者们可以自由发挥，以图文并茂的方式使读者更容易了解文章内容。
  • 我们在每一篇文章上都加上了作者的照片和作者简介。 在简介中可以用几句话来描写作者的生平，经历，兴趣和接触 Linux 的经验。 这些信息可以放在样本文件中的 AboutTheAuthor 後，照片则可放在 AuthorImage 後。
  • 在写作之前请先思考该如何编写您的文件和如何表达您的构想。 一篇好的文章可以让新 Linux 使用者很容易的领会，却又不失老手阅读的兴趣。
  • 一篇文章是一扇知识之窗，是提供读者了解某种领域的门户。 别让这扇门就这麽关著 -- 在您的文章结尾附上一些参考资料， 不管是网上的（URL 连结）或是已出版的书藉皆可。
  • 完成的文章可以送到负责您的语言的编辑，或是直接寄给总编辑 (目前为 guido.socher@linuxfocus.org )。 lf-team 主页上有所有联络人的电子邮件 (E-mail) 地址。 如果您的文章包函了图形或是程式码档案，最好能将其打包成一个 tar.gz 档案，例如在 Linux 下使用下列的语法：
    tar zcvf myart.tar.gz file1 file2 file3...

希望您会喜欢撰写 Linux 的文章。谢谢您的时间、贡献和创造力！ 我们期待您的投稿！
