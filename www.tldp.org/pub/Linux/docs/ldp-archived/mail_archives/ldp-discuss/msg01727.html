Perhaps we should use CVS to handle these problems.  We could specify threee
"revisions"; one for the "current" LDP docs, which should be the default.
The second for "development", for the changes/next version that the author
is working on, and the third for "unmaintained", which could have all of the
documents that have been given up (to public domain?) by their original
authors.  Obviousally not all authors will want to use CVS for their
development work, and some will never have CVS access.  For anyone who
doesn't want to use CVS, or can't, we need to keep ldp-submit around, and
someone will have to volunteer to place those documents into the CVS
repository (that's me, until I decide that I don't want to do it anymore, or
unless there is another volunteer that REALLY wants the job).  As long as
I'm rambling about CVS here, is the mirror on metalab online?
