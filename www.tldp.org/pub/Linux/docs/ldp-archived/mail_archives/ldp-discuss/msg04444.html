Announcement: new listserver
----------------------------
Linuxdoc is happy to annouce new listserver,
which extends our services with almost sure
spam elimination, web archive, on-line searching,
very fast delivery. Thanks to Ibibilo.org for
dedicated machine and network. Thanks to Debian for
maintaining our lists till now.

We have made such e-lists on our server, subscription is open
for all of them:
1) discuss@linuxdoc.org - general developers list
parameters: subscribers post only, please subscribe before posting
2) docbook@linuxdoc.org - docbook related problems
parameters: subscribers post only, please subscribe before posting
3) announce@linuxdoc.org - major announcement, low volume list
parameters: only linuxdoc staff could post, if you want to announce
something important, please mail your proposition to anyone
from linuxdoc staff firstly, their e-mail addresses available
from http://linuxdoc.org/vlist.html
4) cvs-commits@linuxdoc.org - commits to our CVS repository
parameters: only cvs server could post, more info:
http://linuxdoc.org/LDP/LDP-Author-Guide/cvs.html
5) submit@linuxdoc.org - general submits address, list designed for one-way
communication from authors to linuxdoc staff,
but anyone could subsribe to check our hard work :)
[remember that we prefer CVS for new submits!]
parameters: everone could post, so list volume is not known,
spam is possible.

Becouse our new server checks exact Sender of message to prevent spam,
all of you should re-subscribe to new lists by yourself. The old server
will work only to 15th December 2000!
To subscribe please send empty message to:
<list-name>-subscribe@linuxdoc.org, eg. announce-subscribe@linuxdoc.org
Each list is archived in two forms: text files, accessible by e-mail,
and on www: http://list.linuxdoc.org/ (with mysql driven searchable
archive). Listserver commands are available by sending empty message
to <list-name>-help@linuxdoc.org, eg. announce-help@linuxdoc.org.
If you have any questions, please ask our listmaster: ser@linuxdoc.org,
do not bother to check listserver page http://list.linuxdoc.org/ also.
