While you're at it, see if you can get the mailer fixed to handle
Received: headers correctly.

When I read the headers of list messages trying to track & kill the
spammer, or find the open relay he's abusing, the only Received:
headers are:

Received: from murphy.debian.org (murphy.debian.org [216.234.231.6])
        by mail.storm.ca (8.8.8+Sun/8.8.8) with SMTP id BAA03433
        for <sandy@storm.ca>; Tue, 28 Mar 2000 01:04:37 -0500 (EST)
Received: (qmail 8392 invoked by uid 38); 28 Mar 2000 05:44:57 -0000

qmail has not recorded where the message came from and has deleted
any previous Received: headers. It is impossible to track the spam.

I complained about this to postmaster@debian a while back, got some
brain-dead excuse, and gave up.
