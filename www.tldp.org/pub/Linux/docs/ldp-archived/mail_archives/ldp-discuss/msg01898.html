Greg Ferguson <gferg@hoop.timonium.sgi.com> wrote:
> On Apr 4,  7:21pm, Tom Fawcett wrote:
> > Are there instructions anywhere that explain, start to finish, how to
> > install a set of working tools for converting DocBook documents into
> > (tex,pdf,html,txt)?
> 
> This is (still) being worked on. It's badly needed. I believe
> Jorge is going to fold it into his Using DocBook HOWTO (but I
> can't say specifically).
> 
> Currently we have a HOWTO HOWTO, a Using DocBook HOWTO (under
> development), and a Guide Submission HOWTO. I think we need to
> consolidate this into one easily-maintainable Author's Guide
> (or Author's HOWTO).
> 
> Does that seem like the correct approach?

Sounds good to me.  Since DocBook is the future it makes sense to have
these integrated into a single LDP Author's Manual (whatever it ends up
being called).

It might also be nice, following the OSWG's example, to have a single page
or directory on linuxdoc.org containing the set of SGML processing tools
the LDP uses.  This way an author can be assured of having on his/her
machine the same set of tools the LDP uses to process documents, in case
feature/version incompatibilities arise.  Greg, the package listing you
posted a little while ago is pretty much this.
