this is slightly off-topic, but i think on point. for very quick and easy
collaboration on simple documents, tom boutell's openfaq package looks
great. very little author overhead, easy to install and even fairly
secure. plus the boutell code which now puts out html faq pages could be
used to produce simple sgml or better xml. 
http://www.boutell.com/openfaq/source/
this would be a very low tech QC implementation for fast response
documents. authors all do the QC and the work is immediately available.
