Seems like an ancient problem. Some people think that squid.nlanr.net does
constitute a valid URL. But it aint. The valid URL should have been:

	http://squid.nlanr.net/

People tend to forget the PATH at the end just as easy as the protocol
header.

Something proofreaders should take notice of.
