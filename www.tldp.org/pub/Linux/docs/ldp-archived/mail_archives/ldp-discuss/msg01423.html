> > I agree with having a directory for each DocBook HOWTO, but not
> > because it can't be converted to txt. The tag I'm suggesting at the
> 
> Ok
> 
> > Don't deprecate plain text documents!!! They are very useful over slow
> > links or for a quick reference. 
> 
> So you will have a DOCBOOK dir and subdirs for each DocBook document.

Why not convert DocBook docs to plain text and the other formats and
then mix them in with the other directories.  For the time being text
docs are needed that have no pictures so authors should write docs so
as to have all the essential information in a picture also explained
in the text.  The picture would help one understand it but would not
be essential.

If a doc consists of more than one file (e.g. html with PNG files)
then it will appear as a directory next to other docs that are single
files.  I think it's confusing and unnecessary to use separate
directories for docs created from DocBook/LinuxDoc.
