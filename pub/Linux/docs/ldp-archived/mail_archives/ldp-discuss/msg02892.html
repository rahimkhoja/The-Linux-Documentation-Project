I prefer the style of example.sgml.  You put in real data and then the
user changes it.  For example:
<!-- Use this template as a model for you own HOWTO -->
<pubdate>v1.0, 10 June 10, 2000</pubdate> 
		<!-- This is the required LDP version/date format -->

I think that there are too many comments and that much of this info
should be text in the body (sections) of the template.
