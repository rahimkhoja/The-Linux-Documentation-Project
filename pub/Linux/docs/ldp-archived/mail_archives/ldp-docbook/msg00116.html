"Anthony E. Greene" wrote:
> 
> Some versions of IE4 cannot display a PNG image by itself.

What versions or IE4?

IMHO the format LDP uses in HTML documents should be "good-old"
(and unfortunately patented) GIF.

The usability of documentation should not be dictated by technology
nor by politics.  JPEG is bad because it looses data and PNG is bad
because not everyone will see it(?).

So, my suggestion is to take a deep breath, buy some GIF compression
utility (xv comes to mind) and make the GIFs for the docs.  That way
end-users will be happy, and the authors won't need to pretend that
we have a good-for-all substitute for GIFs (someday this will be PNG,
but obviously we're not there yet!).
