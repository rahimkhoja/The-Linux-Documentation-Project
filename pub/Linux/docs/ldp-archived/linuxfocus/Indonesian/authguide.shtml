Panduan Untuk Penulis

  • Kami menyarankan untuk melakukan pengeditan artikel Anda menggunakan text editor biasa. HTML Editor WYSIWYG bisa juga melakukan pekerjaan kita tetapi biasanya menghasilkan file-file HTML yang "ugly" dengan banyak kode &nbsp; dan non standard atribut tag didalamnya. Jika Anda belum pernah menulis halaman HTMLI sebelumnya, maka ini akan menjadi kesempatan terbaik untuk memulai. Lihatlah pada panduan sederhana HTML. Beberapa tag - tag dasar HTML sudah cukup untuk menulis sebuah artikel.

  • Bagaimanapun cara Anda mengedit artikel Anda, Anda mesti menggunakan pola kami berupa template (new001.meta.shtml). Ini adalah file HTML sederhana dengan beberapa struktur didalamnya. Setelah heading ArticleBody Anda bisa menuliskan artikel sesuka Anda. Simpanlah file template ini disk dan mulailah mengedit.

  • Jika judul dari artikel menggambarkan isinya, kami mengharapkan adanya ilustrasi didalam artikel dan beberapa gambar pada tubuh artikel. Penulis bebas untuk menggunakan gambar-gambar untuk memperjelas ide-idenya, mengorganisasikan konsep-konsepnya atau untuk memotivasi para pembacanya.

  • Setiap artikel di majalah kami berisikan foto dari penulis dan sedikit biografi pribadi . Biografi ini bisa beberapa paragrap kecil dan bertujuan agar penulis berkesempatan untuk berbicara tentang sejarah mereka , latar belakang profesi, hobby, hubungan mereka ke Linux, dll. Informasi ini termasuk pada template setelah heading AboutTheAuthor. Gambar dari penulis terletak setelah AuthorImage.

  • Sebelum menulis sebuah artikel baru berfikirlah sedikit tentang struktur dari artikel Anda dan pesan yang akan Anda sampaikan pada para pembaca. Artikel terbaik adalah artikel yang bisa dipahami oleh para pengguna Linux pemula dan masih menarik untuk para pengguna Linux yang sudah berpengalaman.

  • Sebuah artikel adalah jendela menuju ke sebuah lapangan ilmu pengetahuan. Jangan biarkan jendela ini tertutup. Termasuk referensi di akhir dari sebuah artikel, baik publikasi di Web (URL-URL) dan sumber-sumber yang bisa dicetak.

  • Pengalaman menunjukkan bahwa kebanyakan pembaca lebih menyukai artikel yang dapat selesai dibaca kurang dari 20 menit. Jika Anda mempunyai artikel yang lumayan panjang maka pertimbangkanlah untuk membagi artikel besar tersebut menjadi 2 artikel yang lebih kecil (contoh artikel1: Perkenalan, Instalasi, contoh kecil, artikel2: fitur-fitur yang lebih dalam beserta contohnya). Sebagai panduan penulisan: 1500 sampai 3500 kata. Tidak lebih. Anda bisa menggunakan perintah UNIX wc -w untuk menghitung kata-kata tersebut.

  • Artikel bisa and kirimkan kepada editor untuk bahasa Anda sendiri atau langsung ke pimpinan editor, (sekarang ini Guido Socher ). Halamanlf-team berisi daftar dari kontak person kami beserta alamat email-nya. Sangat disarankan artikel Anda dan gambar yang ada didalamnya, contoh code dll... disatukankan kedalam satu paket berupa arsip tar.gz. Perintah untuk melakukan hal ini pada Linux adalah:
    tar zcvf myart.tar.gz file1 file1 file3...

  • Anda mesti berotonomi penuh terhadap tulisan/artikel Anda. Artinya: artikel tersebut asli dan tidak boleh hasil tulisan orang lain. Jika Anda mempublikasikan nformasi mengenai proyek atau software atau apa saja yang tidak mempunyai lisensi copyright bebas/terbuka maka Anda mesti mendapatkan izin dari pihak bersangkutan untuk menulis hal tersebut. Contoh: Anda tidak seharusnya menulis tentang sebuah software baru yang dikembangkan dalam proyek rahasia CIA kecuali Anda mempunyai izin tertulis untuk itu.
