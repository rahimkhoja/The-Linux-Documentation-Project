為了加速文件處理的速度和保持 LinuxFocus 文章的品質及完整性， 我們列出了以下幾點守則，希望想要投稿的作者們能夠參考並盡可能地遵守這些統一的格式。





寫作守則

  • 我們建議您用文字編輯器來編輯您的文章，雖然您也可以用一些『所見即所得』 WYSIWYG 的 HTML 編輯器來寫作，但這些編輯器常在檔案內加入一些 紊亂的 HTML 語法，如一大堆的 &nbsp; 和 <BR> 等等。

  • 不管您決定用什麼編輯您的文章您都應該使用我們的樣本(template.meta.html)。這個檔案只是一個簡單的 HTML 檔案，但包涵了一些結構資訊。您只要把文章放在 ArticleBody 這一個關鍵詞的後面就可以了。

  • 如果文章的主題需要，我們建議作者們在文章中加入一些插圖或圖片， 作者們可以自由發揮，以圖文並茂的方式使讀者更容易了解文章內容。

  • 我們在每一篇文章上都加上了作者的照片和自傳的空間。 在自傳中可以用幾句話來描寫作者的生平，經歷，興趣和與 Linux 的淵緣等等。 這些資訊可以放在樣本文件中的 AboutTheAuthor 後，照片則可放在 AuthorImage 後。

  • 在寫作之前請先思考該如何編寫您的文件和如何表達您的構想。 一篇好的文章可以讓新 Linux 使用者很容易的領會，卻又不失老手閱讀的興趣。

  • 一篇文章可被視為一扇知識之窗，是提供讀者了解某種領域的門戶。 別讓這扇門就這麼關著 -- 您可在您的文章結尾附上一些參考資料， 不管是網上的（URL 連結）或是已出版的書藉皆可。

  • 完成的文章可以送到負責您語言的編輯，或是直接寄給總編輯 (目前為 guido.socher@linuxfocus.org )。 lf-team 網頁上有所有聯絡人的電子郵件 (E-mail) 地址。

  • 如果您的文章包函了圖形或是程式碼檔案，最好能將其打包成一個 tar.gz 檔案，例如在 Linux 下使用下列的語法：
    tar zcvf myart.tar.gz file1 file2 file3...

希望您會喜歡撰寫 Linux 的文章。謝謝您的時間、貢獻和創造力！ 我們期待您的投稿！
