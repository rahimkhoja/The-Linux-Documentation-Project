 Linux Documentation Project Mailing Lists & Feedback 

We welcome your comments on the Linux Documentation Project (LDP); but remember that we are a volunteer organization with limited resources so we can't do everything. Feedback will be sent to a mailing list that is only for discussion of LDP issues, so please don't ask technical or general questions about Linux.

We again wish to stress that these lists are for the discussion of issues related to the LDP; including such topics as: the creation, maintenance, and translation of documents (HOWTOs, FAQs, Guides, etc.); ideas for new documents; issues involving the LDP web site; issues involving SGML processing tools; etc.

For questions of a general Linux nature, may we suggest:

  • Take a look at the Linux Frequently Asked Questions with Answers (FAQ).
  • Examine the General Information or our New Users and Help Sites links pages.
  • Use the mailing list of a local Linux User Group.
  • Utilize a site such as JustLinux, linuxforums.org, or linuxquestions.org.
  • Try a search of the LDP.

If you are clear on this policy, we welcome and appreciate your feedback.


Mailing Lists

There are a few mailing lists you may wish to subscribe to, in order to be involved and stay informed about the work being done by the LDP (to subscribe, click on the link for each and remove ".NOSPAM" prior to sending or see the instructions below):

  1. discuss@en.tldp.org - general discussion list about documenting software and hardware.
    Parameters: subscribers post only, please subscribe before posting.

  2. docbook@en.tldp.org - DocBook (SGML) and document publishing related problems.
    Parameters: subscribers post only, please subscribe before posting

  3. announce@en.tldp.org - major announcements and document updates posted.
    Parameters: a low volume list; only LDP staff can post. If you wish to announce something important, please mail your proposal to someone from the LDP staff first. Their email addresses are available from the volunteers list.

  4. i18n@tldp.org - Discussion about translation projects for the LDP.
    Parameters: subscribers post only, please subscribe before posting

  5. editors@en.tldp.org - LDP Review team editors and reviewers list.
    Parameters: subscribers post only, please subscribe before posting

  6. feedback@en.tldp.org - provide feedback to the LDP.
    Parameters: everyone can post, so list volume is not known; spam is possible.

  7. submit@en.tldp.org - general (document) submits address; list designed for one-way communication from authors to LDP staff for submission of new or updated document (but anyone can subscribe). Remember that we prefer GitHub for new submits, followed by a message to submit@en.tldp.org!
    Parameters: everyone can post, so list volume is not known; spam is possible.

Because our new server checks the exact sender of a message (to prevent spam), you need to subscribe to these lists as "yourself" (from the mail account you will post or receive from).

To subscribe, send an empty message to: <list-name>-subscribe@en.tldp.org (eg. announce-subscribe@en.tldp.org) or click on the links provided above.

To unsubscribe, send an empty message to: <list-name>-unsubscribe@en.tldp.org (eg. announce-unsubscribe@en.tldp.org).

Each list is archived in two forms: text files (accessible by e-mail), and from http://lists.tldp.org/ (with a mysql-driven searchable archive). Listserver commands are available by sending an empty message to <list-name>-help@en.tldp.org (eg. announce-help@en.tldp.org).

If you have any questions, please e-mail our listmaster: ser@tldp.org.
